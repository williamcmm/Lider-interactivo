# LEEME - Guía de instalación y activación de sincronización en tiempo real

Este proyecto está listo para que cualquier persona pueda compartir un enlace y que los demás vean el contenido en tiempo real, usando Firebase/Firestore. Solo sigue estos pasos y recomendaciones:

## 1. ¿Qué está implementado?
- Botón Compartir: Permite seleccionar qué contenido compartir (lectura, diapositiva, ayudas, notas).
- Creación de sesión compartida: Al compartir, se crea una sesión en Firestore (`sharedSessions`) con los datos seleccionados y se genera un enlace único.
- Enlace compartido: El enlace incluye el ID de la sesión. Los participantes que abran ese enlace verán el contenido sincronizado en tiempo real.
- Página `shared-slide`: Está preparada para leer el ID de sesión y conectarse a Firestore. Cuando el presentador cambie el contenido, todos los participantes verán el cambio automáticamente (sin recargar).

## 2. ¿Qué debes hacer para que funcione en producción?

## 3. ¿Cómo funciona el flujo de compartir?
- El presentador selecciona el contenido y comparte el enlace.
- Se crea una sesión en Firestore y se genera un enlace único.
- Los participantes abren el enlace y ven el contenido en tiempo real.
- Si el presentador cambia el fragmento o el tipo de contenido, todos los participantes ven el cambio al instante.
